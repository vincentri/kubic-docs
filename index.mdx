## What is Kubic?

Kubic is an open-source Kubernetes management platform that simplifies operations across multiple Kubernetes clusters. Whether you're managing clusters manually or provisioning them on AWS, Kubic provides a unified control plane to deploy applications, manage resources, and optimize costs.

## Architecture Overview

Kubic follows a clean architecture pattern with a control plane that manages multiple Kubernetes clusters:

```
┌──────────────────────────────────────────────────┐
│  Kubic Control Plane                             │
│                                                   │
│  ┌────────────────┐        ┌──────────────────┐ │
│  │  Next.js App   │◄──────►│  Go API (Gin)    │ │
│  │  - Dashboard   │  REST  │  - Swagger docs  │ │
│  │  - TypeScript  │        │  - Clean arch    │ │
│  │  - shadcn/ui   │        │  - Services      │ │
│  └────────────────┘        └────────┬─────────┘ │
│                                     ↓            │
│                            ┌──────────────────┐ │
│                            │  PostgreSQL      │ │
│                            │  - Clusters      │ │
│                            │  - Applications  │ │
│                            │  - Templates     │ │
│                            └──────────────────┘ │
└──────────────────────────────┬───────────────────┘
                               ↓ (kubectl + kubeconfig)
                    ┌──────────┴──────────┐
                    ↓                     ↓
            ┌───────────────┐     ┌───────────────┐
            │ Manual Cluster│     │ AWS Managed   │
            │ (Any K8s)     │     │ (EC2 + K3s)   │
            │ - Register    │     │ - Provision   │
            │   kubeconfig  │     │ - Stop/Start  │
            │               │     │ - Auto HTTPS  │
            └───────────────┘     └───────────────┘
```

## Use Cases

- **Multi-Environment Management**: Manage development, staging, and production clusters from one place
- **Rapid Application Deployment**: Deploy applications using templates without writing Helm charts manually
- **Cost-Effective Development**: Provision temporary clusters on AWS and stop them when not in use
- **Centralized Operations**: Monitor and manage resources across all your Kubernetes clusters
- **Template Library**: Build and share application templates for your team

## Tech Stack

- **Backend**: Go + Gin + GORM + PostgreSQL
- **Frontend**: Next.js 14 (App Router) + TypeScript + Tailwind + shadcn/ui
- **Deployment**: Helm for application packaging
- **Infrastructure**: AWS EC2 provisioning with Elastic IPs
- **Ingress**: Traefik with automatic Let's Encrypt certificates
