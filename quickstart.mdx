## Prerequisites

Before you begin, ensure you have the following installed:

- **Docker** (version 20.10 or later)
- **Docker Compose** (version 2.0 or later)
- **Git** (to clone the repository)

<Info>
You can verify your Docker installation by running `docker --version` and `docker compose version`.
</Info>

## Step 1: Clone the Repository

```bash
git clone https://github.com/vincentri/kubic.git
cd kubic
```

## Step 2: Environment Variables Setup

Create a `.env` file in the project root directory. You can start with the defaults for local development, but you may want to customize certain values.

### Database Configuration

```bash
# PostgreSQL Database Settings
DB_HOST=postgres              # Database hostname (use 'postgres' for Docker Compose)
DB_PORT=5432                 # Database port
DB_USER=postgres             # Database username
DB_PASSWORD=postgres         # Database password (change in production!)
DB_NAME=kubic                # Database name
DB_SSLMODE=disable           # SSL mode (disable for local, require for production)
```

### Server Configuration

```bash
# Backend API Server
PORT=8080                     # Port for the backend API server
```

### Redis Configuration

```bash
# Redis Cache/Queue Settings
REDIS_HOST=redis             # Redis hostname (use 'redis' for Docker Compose)
REDIS_PORT=6379              # Redis port
REDIS_PASSWORD=              # Redis password (empty for local development)
REDIS_DB=0                   # Redis database number
```

### Encryption Key

<Warning>
**Security Warning**: The encryption key is used to encrypt sensitive data (like kubeconfigs) in the database. In production, you MUST use a strong, randomly generated 32-byte key. Never use the default value in production!
</Warning>

```bash
# Encryption Key (32 bytes required)
ENCRYPTION_KEY=12345678901234567890123456789012  # Default for local dev only!
```

To generate a secure encryption key:

```bash
# Using OpenSSL
openssl rand -hex 32

# Or using Python
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### AWS Configuration (Optional)

These are only required if you want to use AWS cluster provisioning features:

```bash
# AWS Credentials (optional - only for AWS cluster provisioning)
AWS_REGION=us-east-1                    # AWS region for EC2 instances
AWS_ACCESS_KEY_ID=                       # AWS access key ID
AWS_SECRET_ACCESS_KEY=                    # AWS secret access key
```

<Tip>
If you're using AWS CLI with profiles, you can set `AWS_PROFILE` instead of access keys. The backend will use the default AWS credentials chain.
</Tip>

### Frontend Configuration

```bash
# Frontend Settings
NEXT_PUBLIC_API_URL=http://localhost:8080  # Backend API URL
FRONTEND_PORT=3000                         # Frontend port
```

### Worker Configuration

```bash
# Background Worker Settings
WORKER_CONCURRENCY=10  # Number of concurrent background jobs
```

### Complete Example `.env` File

Here's a complete example `.env` file for local development:

```bash
# Database
DB_HOST=postgres
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=kubic
DB_SSLMODE=disable

# Server
PORT=8080

# Redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# Encryption (change in production!)
ENCRYPTION_KEY=12345678901234567890123456789012

# Frontend
NEXT_PUBLIC_API_URL=http://localhost:8080
FRONTEND_PORT=3000

# Worker
WORKER_CONCURRENCY=10

# AWS (optional - only for AWS cluster provisioning)
# AWS_REGION=us-east-1
# AWS_ACCESS_KEY_ID=
# AWS_SECRET_ACCESS_KEY=
```

## Step 3: Start All Services

Navigate to the `docker` directory and start all services:

```bash
cd docker
docker compose up -d
```

Or from the project root:

```bash
docker compose -f docker/docker-compose.yml up -d
```

This command will:
- Start PostgreSQL database
- Start Redis cache/queue
- Run database migrations automatically
- Run database seeders automatically
- Start Backend API server
- Start Frontend dashboard
- Start Worker service for background jobs

<Info>
The `-d` flag runs containers in detached mode (in the background). Remove it if you want to see logs in the terminal.
</Info>

## Step 4: Verify Installation

Check that all services are running:

```bash
cd docker
docker compose ps
```

You should see all services with status "Up" or "healthy":
- `kubic-postgres` (PostgreSQL)
- `kubic-redis` (Redis)
- `kubic-backend` (Backend API)
- `kubic-frontend` (Frontend Dashboard)
- `kubic-worker` (Background Worker)

### View Logs

To view logs from all services:

```bash
cd docker
docker compose logs -f
```

To view logs from a specific service:

```bash
# Backend logs
docker compose logs -f backend

# Frontend logs
docker compose logs -f frontend

# Worker logs
docker compose logs -f worker

# Database logs
docker compose logs -f postgres
```

## Step 5: Access the Platform

Once all services are running, you can access:

- **Frontend Dashboard**: [http://localhost:3000](http://localhost:3000)
- **Backend API**: [http://localhost:8080](http://localhost:8080)
- **Swagger Documentation**: [http://localhost:8080/swagger/index.html](http://localhost:8080/swagger/index.html)
- **Health Check**: [http://localhost:8080/health](http://localhost:8080/health)

## Step 6: Stop Services

To stop all services (keeps data):

```bash
cd docker
docker compose down
```

To stop and remove volumes (resets database):

```bash
cd docker
docker compose down -v
```

<Warning>
Removing volumes will delete all data in PostgreSQL and Redis. Use with caution!
</Warning>

## Troubleshooting

### Services Won't Start

Check the logs to identify the issue:

```bash
cd docker
docker compose logs
```

Common issues:
- **Port conflicts**: Ensure ports 3000, 5432, 6379, and 8080 are not in use
- **Database connection**: Wait for PostgreSQL to be healthy before starting backend
- **Environment variables**: Verify your `.env` file has correct values

### Database Connection Issues

Verify PostgreSQL is healthy:

```bash
docker compose ps postgres
docker compose logs postgres
```

### Reset Everything

If you need to start fresh:

```bash
cd docker
docker compose down -v
docker compose up -d
```

This will:
- Stop all containers
- Remove all volumes (database data)
- Rebuild and start fresh

### Port Already in Use

If you get port conflicts, you can change ports in your `.env` file:

```bash
PORT=8081                    # Change backend port
FRONTEND_PORT=3001           # Change frontend port
DB_PORT=5433                 # Change database port
REDIS_PORT=6380             # Change Redis port
```

Don't forget to update `NEXT_PUBLIC_API_URL` if you change the backend port!
